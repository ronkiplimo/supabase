## Usage

```tsx
import { StateBadge } from 'ui-patterns/StateBadge'
```

```tsx
<StateBadge state="success" />
```

## API Reference

| Prop        | Type                                                                          | Default | Description                                                            |
| ----------- | ----------------------------------------------------------------------------- | ------- | ---------------------------------------------------------------------- |
| `state`     | `'success' \| 'failure' \| 'pending' \| 'skipped' \| 'enabled' \| 'disabled'` | —       | Semantic state. Determines both the icon and the badge tone.           |
| `children`  | `React.ReactNode`                                                             | —       | Optional label override. Keeps the semantic icon and colour treatment. |
| `className` | `string`                                                                      | —       | Additional classes applied to the root wrapper.                        |

## Guidance

Use `StateBadge` for short operational states where icon plus label improves scanability, such as delivery results or enabled and disabled status.

Keep the text to one or two words. For broader product categories or generic contextual metadata, keep using the `Badge` atom instead.
